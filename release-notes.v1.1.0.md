# Catalog v1.1.0 Release Notes

## API Updates

- Biological context full text search and query by biosample term (DSERV-1140)
  - Adds full-text search on biological context and enables filtering by biosample term across relevant endpoints
- `/variants/phenotypes` now returns `biological_context` (DSERV-1226)
- Add `source` field to `variants/disease` and `disease/variants` responses (DSERV-1182)
- Updated protein response format (DSERV-1189)
- Add `strand` to transcript API response (DSERV-1202)
- API refactor for `genes/coding-variants/scores` endpoint (DSERV-1168, DSERV-1160)
  - Improved structure and added ordering by variant position
- Filter out `aapos = -1` entries from `genes/coding-variants/scores` (DSERV-1219)
- Add `method` filter to data-centric edge endpoints (DSERV-1121)
- Add type filter to `genes-structure` API (DSERV-1198)
- API update for BioGrid/IntAct `genes/genes` and `proteins/proteins` endpoints (DSERV-1193)
- Fix `/variants/proteins` API method enum value from "allele specific binding" to GVATdb (DSERV-1207)
- Fix API for MPRA-like variant annotation data (DSERV-1201)
- Remove Bluestarr enum from `variants/genomic_elements` endpoint (DSERV-1224)
- Use shared helper function for API enum generation instead of hardcoded values (DSERV-1213)
- API fixes for `variants/genes`, `genes/variants`, `genomic_elements/genes`, `variants/phenotypes`, `genes/coding-variants/scores`, `coding-variants/phenotypes`, and MPRA endpoints (DSERV-1144, DSERV-1146, DSERV-1150, DSERV-1151, DSERV-1166, DSERV-1178)
- Fix production query returning Output Validation Failed, which was breaking the interface (DSERV-1122)
- Fix slow variant query (DSERV-1027)
- Updated transcripts and protein-related API after data reload (DSERV-1131)
- Cross-checked and updated all enums in Swagger API (DSERV-1222)
